# Gap Fixes — @-Symbol Resolvers + Composer Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire up the @Files, @Folders, @Code, @Git context resolvers (currently stubs) and route Composer mode messages to the MultiAgentRunner workflow.

**Architecture:** The `ContextResolver` class in `src/agent/context-resolver.ts` accepts injected deps at construction time — we extend the `ContextResolverDeps` interface with four optional callbacks (`fileReader`, `getEditorContext`, `runShellCommand`, `workspaceSymbols`) and implement the resolvers to call them. `extension.ts` wires VS Code APIs into each callback. `chat-view-provider.ts` gets a one-line fix so `@Codebase` with no argument uses the full message. Composer mode routing is wired in `chat-view-provider.ts` by checking `this.agent.getMode()` and dispatching to `champ.runMultiAgent`.

**Tech Stack:** TypeScript, VS Code Extension API (`vscode.workspace.fs`, `vscode.window.activeTextEditor`, `vscode.commands.executeCommand`), Node.js `child_process.exec`, Vitest.

  workspaceRoot: string;

  };
  /** Web search tool (from Phase 9+). */

      args: Record<string, unknown>,
    ): Promise<{ success: boolean; output: string }>;
  };
  /**
   * File system access. Optional — if absent, @Files and @Folders return
   * a short placeholder so the resolver degrades gracefully in tests.

    readFile(absPath: string): Promise<string>;
    readdir(absPath: string): Promise<Array<[string, "file" | "directory"]>>;

   * Used by the @Code resolver.

  getEditorContext?: () => {

    filePath: string;
    language: string;

  /**
   * Run a shell command in the workspace root and return stdout.

  /**

```typescript
  private async resolveContextReferences(text: string): Promise<string> {
    if (!this.contextResolver) return text;

    const refs = this.contextResolver.parseReferences(text);
    if (refs.length === 0) return text;

    // For bare @Codebase with no explicit query (e.g. "How does auth work?
    // @Codebase"), use the full message stripped of the @Codebase token so
    // semantic search has something to search for.
    const strippedText = text.replace(/@Codebase\b/g, "").trim();
    const enrichedRefs = refs.map((ref) =>
      ref.type === "codebase" && !ref.value.trim()
        ? { ...ref, value: strippedText }
        : ref,
    );

    let resolved: Array<{ type: string; label: string; content: string }>;
    try {
      resolved = await this.contextResolver.resolve(enrichedRefs);
    } catch {
      return text;
    }

    if (resolved.length === 0) return text;

    const sections = resolved
      .map((r) => `--- ${r.label} ---\n${r.content}`)
      .join("\n\n");

    return `${text}\n\n# Referenced context\n\n${sections}`;
  }
```
