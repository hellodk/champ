# Sprint K: Onboarding & Model Picker — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a first-run wizard, always-visible model picker chip, session token/cost counter, and mode-aware empty state to Champ's chat interface.

**Architecture:** First-run state is tracked via VS Code globalState ('champ.firstRunComplete'). The model picker chip is rendered inline in the chat header area of main.js, using existing ProviderStatusMessage data. Token totals accumulate in extension.ts and are broadcast as SessionTokenUsageMessage after each streamEnd. Empty state content is mode-specific, defined as a constant map in main.js.

**Tech Stack:** TypeScript (extension host), Vanilla JS (webview), CSS variables (VS Code theme-aware), Vitest (tests)

```typescript
/**
 * token-cost.ts — static per-provider token cost table.
 *
 * Rates are approximate public list prices as of 2026-05.
 * Local providers (ollama, llamacpp, vllm, openai-compatible) return 0.
 * This is intentionally a pure module with no VS Code dependencies so
 * it can be unit-tested without mocks.
 */

interface CostRate {
  /** USD per million input tokens */
  inputPerMillion: number;
  /** USD per million output tokens */
  outputPerMillion: number;
}

const COST_TABLE: Record<string, CostRate> = {
  claude: { inputPerMillion: 3.0, outputPerMillion: 15.0 },
  openai: { inputPerMillion: 5.0, outputPerMillion: 15.0 },
  gemini: { inputPerMillion: 1.25, outputPerMillion: 5.0 },
};

/**
 * Estimate the USD cost for a single LLM turn.
 * Returns 0 for local/unknown providers — cost is not meaningful for them.
 */
export function estimateCost(
  providerName: string,
  inputTokens: number,
  outputTokens: number,
): number {
  const rate = COST_TABLE[providerName.toLowerCase()];
  if (!rate) return 0;
  return (
    (inputTokens / 1_000_000) * rate.inputPerMillion +
    (outputTokens / 1_000_000) * rate.outputPerMillion
  );
}
```

```javascript
  function renderSessionTokenCounter() {
    const { sessionInputTokens, sessionOutputTokens, estimatedCostUsd } = state;

    }
```

```typescript
/**
 * Returns true when neither the new nor legacy key is set, meaning the
 * user has never completed or dismissed onboarding.
 *
 * Accepts any Map-like object so it is unit-testable without VS Code mocks.
 */
export function isFirstRunRequired(
  store: { get: (key: string) => boolean | undefined },
): boolean {
  const complete = store.get(FIRST_RUN_COMPLETE_KEY);
  const dismissed = store.get(ONBOARDING_DISMISSED_KEY);
  return !complete && !dismissed;
}
```
