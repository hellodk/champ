You are a senior VS Code extension architect and TypeScript engineer. Your task is to design and implement a production-grade VS Code extension that delivers a Cursor-like AI coding experience using LOCAL LLMs (llama.cpp / Ollama), with multi-agent orchestration, file upload support, and strong safety controls.

The system must be modular, extensible, and optimized for constrained hardware (Apple Silicon 16GB RAM).

# 🤖 MULTI-AGENT ARCHITECTURE

Design a coordinated multi-agent system with clear responsibilities.

## Agents:

### 1. Planner Agent

* Breaks user request into steps
* Outputs structured execution plan

### 2. Code Agent

* Performs code generation/modification
* Produces diffs

### 3. Reviewer Agent

* Reviews generated changes
* Checks correctness, style, and safety

### 4. Validator Agent

* Runs:

  * lint
  * typecheck
  * tests (if available)
* Can reject or request fixes

### 5. Context Agent

* Retrieves relevant code via:

  * embeddings (vector search)
  * lexical search (ripgrep)

### 6. File Agent

* Handles uploaded files
* Parses, chunks, and indexes content

---

## Agent Coordination Layer

Implement an orchestration engine:

* Central controller (Agent Orchestrator)
* Supports:

  * sequential workflows
  * conditional branching
  * retries with modified parameters
* Shared memory:

  * task state
  * intermediate outputs
* Message passing between agents
* Execution graph (DAG-like)

---

## Example Flow

User Query
→ Planner Agent (create plan)
→ Context Agent (fetch context)
→ Code Agent (generate changes)
→ Reviewer Agent (review)
→ Validator Agent (validate)
→ User Approval
→ Apply changes
