How much human approval during a workflow run?

Enterprise-grade means controllable. This defines when the system pauses for human review vs runs autonomously.

A — Autopilot

Runs all agents to completion automatically. Only control is Stop. File diffs shown after completion — accept/reject the full result.

Pros

  • Fastest — zero interruptions
  • Good for trusted workflows

Cons

  • Can't course-correct mid-run
  • May apply changes you don't want

B — Gate Before Writes (Recommended)

Read-only agents (planner, context) run automatically. A gate pauses before any agent that writes files. You see what's planned, then approve or stop.

Pros

  • Safe by default — no surprise edits
  • Read steps don't slow you down
  • Industry standard for AI agents

Cons

  • One extra click per write agent

C — Full Step-by-Step Control

Each agent pauses after the previous completes. You explicitly approve each step: Run, Skip, or Stop. Full control, maximum safety.

Pros

  • Maximum control and auditability
  • Can skip any step
  • Best for learning or auditing

Cons

  • Slow — lots of clicks
  • Defeats the purpose of automation

D — Per-Run Mode Switch

User picks a mode before each run: Auto (no gates), Safe (gate before writes), Audit (approve every step). Stored as default in config.

Pros

  • Flexible for different contexts
  • Trusted pipelines can go fully auto
  • Sensitive projects stay in Audit mode

Cons

  • More UI complexity
  • Teams need to agree on defaults

For enterprise: B (gate before writes) is the safe default. D (mode switch) adds enterprise flexibility on top.
