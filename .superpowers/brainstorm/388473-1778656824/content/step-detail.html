A — Status Dashboard

Clean pipeline view: name, status icon, duration. No output shown inline — click a step to expand in a side pane.

Pros

  • At-a-glance overview
  • Minimal noise
  • Good for long pipelines

Cons

  • Need extra click to see output
  • Less context when something fails

B — Expandable Steps (Recommended)

Each step has a collapse toggle. Completed steps collapsed by default, current step expanded showing live output. Click any step to expand/collapse.

Pros

  • Progressive disclosure — detail on demand
  • Current step always visible
  • Industry standard (CI/CD pipelines, GitHub Actions)

Cons

  • Slightly more complex to build

C — Rich Detail with Live Diff

Two-panel layout: step list on left, live output + file diffs on right. Shows actual code changes as they happen. Accept/reject per file.

Pros

  • Most powerful — see exactly what changes
  • Accept/reject individual file changes
  • Full Cursor-like experience

Cons

  • Significantly more complex to build
  • Requires diff rendering infrastructure
  • Can be overwhelming

B is the industry-standard pattern (GitHub Actions, Jenkins, Vercel). C is the power-user experience. A is the simplest.
